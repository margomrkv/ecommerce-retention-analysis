# Продуктовая аналитика маркетплейса Olist

**Автор:** [Маргарита Маркова](https://github.com/margomrkv)

Финальный проект по блоку **«Продуктовая аналитика»** (Karpov Courses). Работа выполнена как самостоятельное исследование бразильского e-commerce маркетплейса **Olist** на открытых данных и оформлена в формате портфолио для ревьюеров.

> Это не тестовое задание на отбор, а итоговый проект учебного модуля — пример того, как я подхожу к продуктовым задачам: от постановки вопроса и EDA до метрик, гипотез и рекомендаций для продакта.

---

## Контекст задачи

Стартап развивает новый маркетплейс с товарами из Бразилии. Выручка несколько месяцев не растёт. Продакт-менеджер даёт свободу действий при двух условиях:

- метрики должны расти;
- решения не должны ухудшать пользовательский опыт.

**Роль:** аналитик данных, который помогает понять, где «узкое место» продукта и что делать дальше.

---

## Что сделано

| № | Задача | Подход |
|---|--------|--------|
| 1 | Месячный retention в оформление заказа | Когортный анализ, heatmap, z-тест долей |
| 2 | Оценка product/market fit | Кривая PMF по медианному retention |
| 3 | 5 ключевых метрик для роста прибыли | GMV, активные покупатели, конверсия, repeat rate, AOV — с визуализацией |
| 4 | Приоритизация гипотез | Фреймворк ICE (3 гипотезы) |
| 5 | Метрики для A/B-теста | Целевая (GMV), прокси (2-й заказ), guardrail (доля отмен) |
| 6 | Итоговый отчёт для продакта | Выводы и план действий |

---

## Ключевые выводы

- **Retention критически низкий:** медианный retention 1-го месяца ≈ **0,5%**; ~**97%** покупателей не возвращаются после первой покупки.
- **Product/market fit не подтверждён:** на кривой PMF нет устойчивого «плато» — продукт не формирует привычку повторных покупок.
- **Бизнес зависит от притока новых клиентов**, а не от удержания — риск для юнит-экономики при масштабировании маркетинга.
- **Приоритетная гипотеза (ICE):** новый способ оплаты для роста повторных заказов — требует проверки через A/B-тест.
- **Рекомендация:** не масштабировать привлечение, пока не растут retention и конверсия во 2-й заказ.

Подробная логика, графики и расчёты — в ноутбуке [`olist_marketplace_product_analytics.ipynb`](./olist_marketplace_product_analytics.ipynb).

## Данные

Используются три таблицы из публичного датасета **[Brazilian E-Commerce Public Dataset by Olist](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)** (Kaggle):

| Файл | Содержание |
|------|------------|
| `olist_customers_dataset.csv` | Пользователи (город, штат, уникальный ID) |
| `olist_orders_dataset.csv` | Заказы (статусы, даты воронки) |
| `olist_order_items_dataset.csv` | Позиции заказов (цена, продавец, товар) |

Период данных: **2016–2018**.

## Ограничения анализа

- Retention считается по заказам со статусом `delivered` — отмены и незавершённые заказы в метрику не входят.
- Колонка `freight_value` в учебном описании названа «весом», но в открытых источниках по Olist обычно трактуется как **стоимость доставки**; в работе не использовалась.
- Осень 2018 года — возможный артефакт выгрузки (резкий спад заказов и рост отмен); требует отдельной проверки.
